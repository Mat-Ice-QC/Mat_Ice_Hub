# Nettoyage de disques

Ce guide explique comment nettoyer en toute sécurité les anciennes partitions, les arrays RAID MD, les pools ZFS et les volumes LVM d'un disque donné sur un système Linux.

---

## 1. Identifier le disque

Avant d'effectuer tout changement, identifiez le disque cible. Remplacez `/dev/sdX` par votre disque réel.

```bash
# Lister tous les disques
lsblk

# Afficher les informations détaillées du disque
sudo fdisk -l
```

---

## 2. Supprimer les partitions existantes

Vous pouvez effacer toutes les partitions du disque avec `wipefs` ou `fdisk`.

```bash
# Afficher les signatures de systèmes de fichiers existantes
sudo wipefs /dev/sdX

# Supprimer toutes les signatures de systèmes de fichiers
sudo wipefs -a /dev/sdX

# Optionnellement, supprimer les partitions via fdisk
sudo fdisk /dev/sdX
# - Utilisez 'd' pour supprimer les partitions
# - Utilisez 'w' pour écrire les changements
```

>  Attention : Cela effacera toutes les données du disque.

---

## 3. Supprimer les métadonnées RAID MD

Si le disque faisait partie d'un RAID logiciel, effacez son superbloc RAID.

```bash
# Examiner les métadonnées RAID
sudo mdadm --examine /dev/sdX

# Supprimer le superbloc RAID
sudo mdadm --zero-superblock /dev/sdX
```

> Cela garantit que le disque n'est plus reconnu comme faisant partie d'un array RAID.

---

## 4. Supprimer les métadonnées ZFS

Si le disque faisait partie d'un pool ZFS, effacez les étiquettes ZFS.

```bash
# Lister les pools ZFS
sudo zpool list

# Vérifier à quel pool le disque appartient
sudo zpool status

# Exporter le pool (s'il est encore actif)
sudo zpool export <nom_du_pool>

# Détruire le pool ZFS (si vous voulez le supprimer entièrement)
sudo zpool destroy <nom_du_pool>

# Effacer les étiquettes ZFS du disque
sudo wipefs -a /dev/sdX
```

> ZFS stocke ses métadonnées dans les premiers et derniers secteurs du disque, donc les effacer supprime la référence au pool ZFS.

---

## 5. Supprimer les métadonnées LVM

Si le disque faisait partie d'une configuration LVM, supprimez sa configuration LVM.

```bash
# Lister les volumes physiques LVM
sudo pvs

# Supprimer le volume physique
sudo pvremove /dev/sdX

# Optionnellement, supprimer les groupes de volumes qui sont uniquement sur ce disque
sudo vgdisplay
sudo vgremove <nom_vg>

# Supprimer les volumes logiques sur ce disque
sudo lvdisplay
sudo lvremove /dev/<nom_vg>/<nom_lv>
```

> Cela garantit que le disque n'est plus reconnu comme faisant partie d'un groupe de volumes LVM.

---

## 6. Optionnel : Effacement sécurisé du disque

Si vous voulez effacer complètement toutes les données (y compris les restes de RAID, ZFS ou LVM), vous pouvez utiliser `dd` ou `shred` :

```bash
# Écrire des zéros sur tout le disque
sudo dd if=/dev/zero of=/dev/sdX bs=1M status=progress

# OU effacement sécurisé
sudo shred -v /dev/sdX
```

> Cela peut prendre beaucoup de temps pour les gros disques.

---

## 7. Vérifier que le disque est propre

```bash
# Lister les partitions
lsblk

# Afficher les signatures de systèmes de fichiers
sudo wipefs /dev/sdX
```

Le disque devrait maintenant être propre et prêt pour une nouvelle utilisation.
