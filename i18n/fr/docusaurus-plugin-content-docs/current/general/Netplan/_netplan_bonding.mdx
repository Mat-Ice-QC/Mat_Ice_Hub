      
            # Agrégation d'interfaces

            L'**agrégation réseau** (aussi appelée **link aggregation**) combine plusieurs interfaces réseau en une seule interface logique pour augmenter le débit, améliorer la redondance, ou les deux.  
            Si une interface tombe en panne, le trafic bascule automatiquement vers les autres, assurant la continuité du réseau.

            Avec **Netplan**, la configuration d'une interface agrégée implique trois étapes principales :

            1. **Désactiver** la configuration individuelle (comme le DHCP) sur les interfaces physiques que vous souhaitez agréger.  
            2. **Définir** une interface bond qui liste ces interfaces physiques comme ses membres.  
            3. **Configurer** les paramètres IP (DHCP ou statique) sur l'**interface bond**, pas sur les interfaces physiques.

            ---

            ## Exemple 1 : Bond avec DHCP

            Cet exemple crée un bond nommé `bond0` en utilisant le **mode balance-rr** (tourniquet), et attribue son adresse IP via **DHCP**.  
            Il désactive le DHCP sur les interfaces physiques `enp3s0` et `enp4s0`.

            **Fichier :** `/etc/netplan/50-bonding.yaml`

            ```yaml
            network:
            version: 2
            renderer: networkd
            ethernets:
                enp3s0:
                dhcp4: false
                dhcp6: false
                enp4s0:
                dhcp4: false
                dhcp6: false

            bonds:
                bond0:
                interfaces: [enp3s0, enp4s0]
                parameters:
                    mode: balance-rr
                    mii-monitor-interval: 100
                dhcp4: true
            ```

            ### Étapes d'application

            1. Enregistrez le fichier de configuration ci-dessus dans `/etc/netplan/50-bonding.yaml`.  
            2. Appliquez la configuration :  
            ```bash
            sudo netplan apply
            ```
            3. Vérifiez l'interface bond :  
            ```bash
            ip a
            ```
            Vous devriez voir `bond0` avec une adresse IP obtenue via DHCP.  

            ---

            ## Exemple 2 : Bond avec IP statique

            Cet exemple attribue une **adresse IP statique** à l'interface agrégée au lieu d'utiliser le DHCP.

            **Fichier :** `/etc/netplan/50-bonding.yaml`

            ```yaml
            network:
            version: 2
            renderer: networkd
            ethernets:
                enp3s0:
                dhcp4: false
                dhcp6: false
                enp4s0:
                dhcp4: false
                dhcp6: false

            bonds:
                bond0:
                interfaces: [enp3s0, enp4s0]
                parameters:
                    mode: active-backup
                    primary: enp3s0
                    mii-monitor-interval: 100
                dhcp4: false
                addresses:
                    - 192.168.1.50/24
                routes:
                    - to: default
                    via: 192.168.1.1
                nameservers:
                    addresses:
                    - 8.8.8.8
                    - 8.8.4.4
            ```

            ### Explication

            - `mode: active-backup` — Une seule interface est active à la fois. Si la principale (`enp3s0`) tombe en panne, une autre interface prend le relais automatiquement.  
            - `mii-monitor-interval: 100` — Vérifie l'état du lien toutes les 100 ms.  
            - `addresses`, `routes` et `nameservers` — Configurent les paramètres IP statiques pour l'interface agrégée.

            ---

            ### Test et vérification

            Avant d'appliquer définitivement, testez votre configuration :
            ```bash
            sudo netplan try
            ```

            Appliquez définitivement :
            ```bash
            sudo netplan apply
            ```

            Vérifiez l'état du bond :
            ```bash
            cat /proc/net/bonding/bond0
            ```

            Vérifiez les interfaces réseau :
            ```bash
            ip a
            ```

            ---

            ### Modes d'agrégation courants

            | Mode | Description |
            |------|--------------| 
            | **balance-rr (0)** | Équilibrage de charge en tourniquet sur toutes les interfaces. |
            | **active-backup (1)** | Une interface active ; les autres en veille. Haute disponibilité. |
            | **balance-xor (2)** | Transmission basée sur un hash (MAC/IP). |
            | **broadcast (3)** | Envoie le trafic sur toutes les interfaces simultanément. |
            | **802.3ad (4)** | Agrégation dynamique de liens IEEE 802.3ad (nécessite le support du switch). |
            | **balance-tlb (5)** | Équilibrage adaptatif de la charge en émission. |
            | **balance-alb (6)** | Équilibrage adaptatif de la charge (émission + réception). |

            ---

            Avec Netplan, l'agrégation d'interfaces fournit à la fois **performance** et **résilience**.  
            Vérifiez toujours votre configuration avec `netplan status` et surveillez la santé du bond via `/proc/net/bonding/bond0`.
